### 3. Split-and-Merge with Worktree Isolation for Batch Migrations
**Source:** https://www.mindstudio.ai/blog/claude-code-split-and-merge-pattern
**Finding-ID:** 2026-04-12-split-merge-worktree-batch
**Category:** PATTERN
**What it does:** Spawns up to 10 sub-agents, each in its own git worktree, to process independent work in parallel. No file conflicts. Perfect for batch migrations, multi-file refactors, or competing approaches to the same problem.
**Implementation:**
1. In your AGENTS.md or sub-agent config, set `isolation: worktree` per sub-agent
2. Each sub-agent gets a full repo copy on its own branch
3. For batch work: divide files into N groups, spawn N agents (max 10), each handles its group
4. Review branches, merge winners. Losers auto-cleanup.
5. Example: 50 files need API pattern update → 5 agents × 10 files each, all parallel
**Risk:** SAFE

**Finding-ID:** 2026-04-12-claudemd-token-cost-pruning

**Category:** PATTERN
**What it does:** Build dedicated sub-agents for tasks you do more than twice a week (PR review, DB seeding, pre-deploy checks) and commit them to `.claude/agents/` with restricted tool access. Turns repetitive workflows into one-command operations.
**Implementation:**
1. Identify top 3-5 repetitive workflows (e.g., PR review, deploy checks, data migration)
2. Create AGENTS.md files in `.claude/agents/` for each — define allowed tools, model, isolation mode

4. Commit to repo so every team member gets the same agents
5. Invoke with `/agent [name]` or programmatically via sub-agent spawning

### 12. MCP Prompt Caching: 50-90% Cost Reduction on Tool Definitions
**Source:** https://sudoall.com/mcp-claude-production-patterns/
**Finding-ID:** 2026-04-12-mcp-prompt-caching-tools
**Category:** PATTERN
**What it does:** Anthropic's prompt caching caches tool definitions and system prompts that don't change between requests. For MCP-heavy setups (like Logan's with multiple servers), this reduces per-request cost by 50-90% on the cached portion.
**Implementation:**
1. Ensure MCP tool definitions are static (don't dynamically regenerate schemas per-request)
2. Place tool definitions and system prompts as early as possible in the prompt (caching is prefix-based)
3. For API usage: set `cache_control` breakpoints after your tool definitions block
4. Monitor cache hit rates via API response headers (`anthropic-cache-read-tokens`)
5. This is automatic in Claude Code — main benefit is for custom API integrations

### 13. Context Buffer: Claude Code Reserves 33K-45K Tokens Internally
**Source:** https://claudefa.st/blog/guide/mechanics/context-buffer-management
**Finding-ID:** 2026-04-12-context-buffer-reserve
**Category:** CONFIG
**What it does:** Claude Code reserves 33K-45K tokens from your context window for internal operations (tool definitions, system prompts, safety). Your actual usable context is ~955K, not 1M. Understanding this prevents unexpected compaction triggers.
**Implementation:**
1. Plan for ~955K usable tokens, not the full 1M
2. Set compaction triggers at ~900K to leave headroom
3. If using custom `/compact` focus strings, account for the buffer in your timing
4. For sub-agents: their usable context is similarly reduced — factor this into task sizing
**Risk:** SAFE
