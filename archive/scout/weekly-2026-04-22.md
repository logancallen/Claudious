### CLAUDE_CODE_SUBPROCESS_ENV_SCRUB — Linux PID namespace isolation for child processes
**Source:** https://www.turboai.dev/blog/claude-code-environment-variables-complete-list
**Finding-ID:** 2026-04-22-subprocess-env-scrub
**Category:** CONFIG
**What it does:** When set, Claude Code launches subprocesses (bash, MCP servers, hooks) in scrubbed env + Linux PID namespaces, preventing leak of API keys / parent shell state into child tools. Hardens against malicious MCPs reading `ANTHROPIC_API_KEY` from `os.environ`.
**Implementation:**
1. Linux/WSL only — does not apply to native Windows shell.
2. On Logan's Mac/WSL boxes: add `CLAUDE_CODE_SUBPROCESS_ENV_SCRUB=1` to `~/.claude/settings.json` env block.
3. Verify by spawning a hook that prints `env | grep ANTHROPIC` — should be absent.
4. Pair with existing `CLAUDE_CODE_MCP_ALLOWLIST_ENV` proposal for layered defense.
**Risk:** TEST-FIRST (may break MCPs that genuinely need parent env passthrough — audit list of 5 active MCPs first)

**Risk:** SAFE (additive guardrail; only blocks runaway loops)

### Tool-name validation guard — production MCP hardening
**Source:** https://sudoall.com/mcp-claude-production-patterns/
**Finding-ID:** 2026-04-22-tool-name-validation-guard
**Category:** PATTERN
**What it does:** Validates every tool name Claude returns against the registered MCP tool list before execution. Catches hallucinated tool calls (Claude inventing a tool that doesn't exist) before they hit the MCP server and 500. Reduces silent retries and confusing error chains.
**Implementation:**
1. Add a hook (PreToolUse) that intercepts `mcp__*` calls.
2. Cross-check name against `claude mcp list --json` output cached at session start.
3. On mismatch: return structured error to Claude ("tool 'mcp__x__foo' not registered; available: [...]"), don't forward to MCP.
4. Pair with existing MCP allowlist proposal.
**Risk:** TEST-FIRST (hook authoring; could accidentally block legitimate calls if MCP list is stale)
