### 11. Subagent Invocation Quality Framework — 4 Required Components
**Source:** https://claudefa.st/blog/guide/agents/sub-agent-best-practices
**Finding-ID:** 2026-04-13-subagent-invocation-quality
**Category:** PATTERN
**What it does:** Most sub-agent failures stem from poor invocations, not execution bugs. A structured invocation protocol with 4 required components (context density, explicit instructions, file references, success criteria) dramatically improves sub-agent output quality.
**Implementation:**
1. Every sub-agent invocation must include: (a) comprehensive context, (b) explicit deliverables, (c) specific file paths, (d) measurable success criteria
2. Bad: "Fix authentication" → Good: "Fix OAuth redirect loop in src/lib/auth.ts where login redirects to /login instead of /dashboard"
3. Use domain-based routing: frontend agent, backend agent, database agent — no file overlap
4. Add invocation quality rules to CLAUDE.md under a `## Sub-Agent Dispatch Rules` section
5. For sequential chains: schema → API → frontend (data structures must exist first)
