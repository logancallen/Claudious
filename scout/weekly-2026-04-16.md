### 2. `--worktree` / `-w` Flag — Native Parallel Agent CLI
**Source:** https://botmonster.com/posts/parallel-ai-development-claude-code-sessions-git-worktrees/
**Finding-ID:** 2026-04-16-worktree-cli-flag
**Category:** COMMAND
**What it does:** `claude --worktree <name>` (or `-w <name>`) auto-creates an isolated git worktree, checks out a new branch (pattern `worktree-{name}` branched from `origin/HEAD`), and launches Claude scoped to that directory. Collapses the entire "create worktree → cd → claude" sequence into a single command per agent. Direct upgrade to the split-and-merge pattern Logan already uses.
**Implementation:**
1. Update Claude Code to latest (flag confirmed in recent versions)
2. Run per terminal: `claude -w api-refactor`, `claude -w ui-polish`, etc.
3. For auto-generated name: `claude --worktree` with no argument
4. Headless with tmux: `claude --worktree <name> --tmux`
5. Practical limit: 5-10 parallel sessions before merge bottleneck and API rate-limit throttling
6. Pro-tier users hit limits in minutes with 2+ parallel sessions — upgrade tier required for scaling beyond casual use
**Risk:** SAFE (additive flag; no config required)
